LLM引导的交通枢纽容量规划：融合文本业务输入的优化框架
未明确披露
点击收起
摘要总结
传统枢纽容量规划模型对定量输入优化有效，但难以消化定性的业务上下文。本文提出一种新框架，使用LLM智能体在自然语言业务描述引导下迭代提出枢纽容量决策。核心机制是链式思维推理协议：LLM构建结构化决策表，将每条上下文项映射为具体容量调整（基于其隐含的变化方向和幅度）。新容量决策通过优化模型的反馈循环验证，优化模型提供路由性能指标以指导智能体选择。在真实美国东南部13枢纽货运网络上，该方法相对于隐藏真实解的最优性差距为2.8%，而传统无文本输入的优化模型差距为11.0%。
核心创新点
  • 提出LLM作为「上下文桥梁」，将定性业务洞察融入运筹学工作流，填补定量优化与定性文本输入之间的鸿沟
  • 链式思维推理协议：LLM自动从自然语言业务描述中提取容量调整信号，构建结构化决策表
  • 反馈循环验证机制：优化模型的路由性能指标反哺LLM，形成迭代收敛的容量规划闭环
  • 13枢纽真实货运网络验证，最优性差距从11.0%降至2.8%，提升显著
解决的问题 / 适用场景
物流/货运网络的枢纽容量规划，特别是需要融合业务人员自然语言描述（如季节性需求波动、区域政策调整、运力限制等定性信息）的运筹优化场景

Don't Blame the Large Language Model: How Scaffolding Evolution Shapes Coding Agent Quality
别怪大语言模型：脚手架演化如何塑造编码智能体质量
未明确披露
点击收起
摘要总结
编码智能体（Coding Agent）依赖「智能体脚手架」（agentic scaffolding）—— 位于开发者与 LLM 之间的中间件层，负责编排系统提示、工具执行、上下文管理和迭代推理循环。然而，尽管脚手架以极高的速度演化（超过每天两个版本、数月内数千个 issue），尚无研究考察这种演化如何影响智能体的有效性（effectiveness）和效率（efficiency）。实践者常在脚手架更新后报告质量回退，却始终将其归因于底层模型而非脚手架本身。本文开展首次受控纵向研究，固定底层模型、仅变化脚手架，评估了 Qwen Code CLI 的 35 个连续版本在 SWE-bench Verified 上的表现，并将质量波动追溯到具体的开发模式与架构组件。
核心创新点
  • 首次将脚手架（而非模型）作为独立变量，进行受控纵向研究，填补了编码智能体质量归因的关键空白
  • 系统性地分析了五个主流开源脚手架（Codex、Qwen Code、Gemini、OpenCode、OpenHands）的发布演化特征，揭示出每天超过两个版本的极端发布速度
  • 在固定 LLM 的前提下，对 Qwen Code CLI 的 35 个连续版本逐一在 50 个 SWE-bench Verified 任务上评测，将质量波动直接关联到具体的 PR 与架构变更
  • 提供了定性的因果证据链，连接个别拉取请求与可测量的质量偏移，为脚手架开发者提供了可操作的改进方向
解决的问题 / 适用场景
LLM 驱动的编码智能体开发与运维。适用于智能体框架维护者、脚手架开发者以及使用编码智能体的工程团队，帮助他们理解脚手架更新对智能体实际解题能力的真实影响，避免将脚手架引入的回退错误归咎于底层模型。

SelfMem: Self-Optimizing Memory for AI Agents
SelfMem：AI Agent 的自优化记忆框架
未明确披露
点击收起
摘要总结
现有 AI Agent 虽然支持越来越长的上下文窗口和工具调用，但仍依赖固定策略的记忆系统（固定存储、检索、摘要机制），跨任务时缺乏灵活性、需要手工调参。SelfMem 受自我改进 AI 启发，提出“授人以渔”的思路——不给模型预设记忆策略或格式，而是提供一个带记忆工具和反馈信号的“记忆环境”，让 Agent 通过探索、评估、试错来自主学习和优化自己的记忆策略。在 BEAM 基准上，SelfMem 在 100K、500K、1M 三种对话规模下，相比最强基线分别提升 48.7%、40.8%、41.9%。问题类型分析显示对多样化记忆需求广泛鲁棒；模型引导的策略优化进一步提升性能。
核心创新点
  • 摒弃预设记忆策略，提出 meta-learning 式的自优化记忆框架，让 Agent 在带记忆工具和反馈信号的环境中自主探索最优策略
  • 在 100K~1M token 的大规模对话上下文中，三项指标均显著超越检索、压缩和现有 Agent 记忆基线（提升 40%~49%）
  • 跨问题类型的鲁棒性分析和模型引导策略优化实验验证了框架的泛化能力和进一步改进空间
解决的问题 / 适用场景
需要长期历史经验积累的 AI Agent 场景，如多轮对话助手、长时任务规划、持续学习的自主 Agent 系统

Refused in Chat, Written in Code: Workflow-Level Jailbreak Construction in IDE Coding Agents
对话中拒绝，代码中编写：IDE 编码智能体中的工作流级越狱构造
未明确披露
点击收起
摘要总结
本文揭示了一个严重的安全漏洞：当前主流大模型在 IDE 编码智能体场景下的安全评估仍沿用聊天式单轮有害提示的范式，而实际的越狱攻击可以通过将有害目标拆解并嵌入正常软件开发流程的多个步骤中完成。研究以 GitHub Copilot 为平台，测试了 Claude Sonnet 4.6、Claude Haiku 4.5、Gemini 3.1 Pro 和 Gemini 3.5 Flash 四个闭包后端。结果表明，在直接对话、CSV 读取和单步代码修复基线中，模型几乎完全拒绝有害请求（816 次测试中仅 8 次成功），但在完整工作流下，816/816 次全部生成了不安全的教学性代码补全。
核心创新点
  • 提出『工作流级越狱构造』这一新攻击范式，将有害目标分解到软件开发工作流的正常步骤中，而非通过单次有害提示实现
  • 系统评估了 Copilot 在三种基线（直接对话/CSV 读取/单步修复）和完整工作流下的安全差异，揭示对话级安全基准的严重膨胀
  • 通过两名专家在严格评分标准下独立确认，816/816 次全部成功的结果有力证明了攻击的通用性和严重性
解决的问题 / 适用场景
解决 IDE 编码智能体在现有聊天式安全评测下被高估安全性的问题。适用于 LLM 安全评测、编码智能体安全审计及防御机制设计，特别是需要跨多轮、跨工作流推理安全的场景。

SABLE: An NDA-Safe Closed-Loop LLM Framework for Analog Circuit Optimization in Industrial EDA Flows
SABLE：工业EDA流程中模拟电路优化的NDA安全闭环LLM框架
未明确披露
点击收起
摘要总结
大型语言模型能够提出电路优化决策，但工业模拟流程无法将代工厂PDK内容、专有原理图、绝对仿真路径或受许可证约束的工具状态暴露给云端。SABLE（Safe Analog Boundary for LLM-driven EDA）是一个NDA安全的闭环框架，让LLM通过Cadence Virtuoso、Maestro和Spectre优化模拟电路，同时仅返回经过清洗的拓扑意图、数值指标、工作点摘要和受限回写状态。框架结合了显式威胁模型、28个受管SKILL入口点白名单、每条返回路径上的PDK/路径/模型清洗、结构化的Maestro设置与回写、包含六种机器可检查停止条件的严格JSON动作合约，以及最优状态保持。在20 GHz LC-VCO调谐曲线任务和两级运放任务上评估了11个LLM检查点，后者更具挑战性，4/11模型在15次迭代预算内通过。
核心创新点
  • 提出NDA安全闭环框架，在LLM驱动EDA中实现敏感工业数据零暴露，通过28个白名单SKILL入口点和三条返回路径清洗机制保障数据安全
  • 设计含六种机器可检查停止条件的严格JSON动作合约，确保LLM操作可验证、可终止
  • 通过消融实验证明每个经清洗的反馈通道（拓扑意图、数值指标、工作点摘要、回写状态）都对优化成功不可或缺
  • 在真实的PVT签核场景上验证框架有效性，揭示LLM一旦需要工具纪律、偏置推理和规格修复能力时模型质量差异显著
解决的问题 / 适用场景
工业EDA中受NDA保护的模拟电路自动化优化，特别适用于代工厂PDK、专有原理图和许可证约束工具无法暴露给云端的环境，可用于LC-VCO调谐曲线优化、两级运放设计等模拟电路PVT签核任务

Reward Lightning：同源偏好蒸馏实现快速视频生成
未明确披露

摘要总结
提出 Reward Lightning 统一框架，在视频扩散模型中同时完成偏好对齐和蒸馏加速。核心思想是同源性（homology）：将对齐目标和蒸馏目标评估在相同的潜在特征空间上，避免传统方法中因表征空间不匹配导致的梯度冲突。首先引入潜在奖励模型（LRM），直接在潜在空间打分而无需解码回像素空间；基于 LRM，同源偏好蒸馏（HPD）复用同一骨干网络联合进行对抗蒸馏和偏好对齐，生成仅需 1-4 步的高保真视频。
