SingGuard-NSFA: Extensible Guardrails for Agentic AI via Generative Reasoning and Real-Time Classification
SingGuard-NSFA：面向智能体 AI 的可扩展护栏——融合生成式推理与实时分类
未明确披露
点击收起
摘要总结
本文提出 nsfaguard，一个面向 Agentic AI 系统的安全护栏框架，抵御 prompt 注入、敏感信息窃取、恶意代码请求、危险工具滥用和资源耗尽等操作威胁。首先构建 NSFA 风险分类法，将 185 种风险变体组织为基于 CIA 三元组（机密性、完整性、可用性）的分层体系，并经三个 OWASP 指南交叉验证。基于该分类法，构建覆盖 133 种语言的大型基准套件，包含 93K+ 专有样本和从五个公开 Agent 安全数据集改编的 3,435 个跨源样本。检测方面提出双模方法：基于 SFT 的生成式推理用于可解释的离线审计，同时冻结骨干网络附加判别式分类头，实现约 50ms 的实时检测。发布 0.8B、2B、4B、9B 四个模型，在专有基准上均达到 ≥94% F1，超出最强竞品 6-12 个绝对百分点。
核心创新点
  • NSFA 风险分类法：首个基于 CIA 三元组的智能体 AI 风险系统化分类框架，涵盖 7 个一级域、28 个二级风险、185 个三级变体，明确区分攻击技术与攻击目标两个正交维度，支持独立扩展。
  • 双模架构：生成式推理（离线审计，可解释链式推理）与判别式分类头（冻结骨干网络，~50ms 实时检测）统一于同一框架，兼顾可解释性与低延迟。
  • 本地可扩展的分类头设计：新增风险只需在冻结骨干网络的 embedding 上训练单个轻量分类头，无需重训骨干网络，且从 5 个分类头扩展到 50,000 个头时延迟仅增加 9ms。
  • 跨语言大规模基准：首个覆盖 133 种语言的 Agent 安全评测套件（93K+ 样本），并可作为其他护栏模型的即插即用增强——在冻结的 Llama Guard 3 上附加 NSFA 分类头即可将其提升至所有外部护栏之首。
解决的问题 / 适用场景
适用于 Agentic AI 系统的安全防护场景，对用户查询（输入护栏）和 Agent 响应（输出护栏）进行单轮文本级风险检测，在 Agent 执行代码、调用 API、操作文件等实际动作之前阻断操作威胁。可部署于离线审计（生成式推理模式，提供可解释的安全判定）和在线实时拦截（分类头模式，~50ms 延迟）两种管道。

Learning Safe Agent Behaviour from Human Preferences and Justifications via World Models
通过世界模型从人类偏好和理由中学习安全智能体行为
南安普顿大学；伦敦国王学院
点击收起
摘要总结
论文提出 DROPJ 框架，解决环境动态未知且无合适奖励函数场景下的智能体安全训练与部署问题。该方法先用离线真实轨迹学习世界模型（可交互的模拟器），让人类在其中生成若干条信息量充分的模拟轨迹，然后从轨迹段对中收集人类的偏好判断及其理由（justification），据此训练奖励模型，最后将奖励模型与世界模型结合，通过模型预测控制（MPC）直接部署智能体。实验表明，在模拟器中一次性生成信息轨迹相比迭代式查询大幅降低了训练计算成本，且偏好类反馈优于稀疏标签，偏好附带的理由能在部署阶段显著提升安全表现。
核心创新点
  • 提出 one-shot 信息轨迹生成策略，用户只需在学到的世界模型中演示一次即可获得高效训练数据，避免了传统迭代式查询的高计算开销
  • 将人类理由（justification）引入离线轨迹段偏好学习，训练奖励模型的同时支持多条安全理由，实现部署阶段按用户指定的安全维度动态调整行为
  • 在无已知奖励函数且动态未知的安全关键场景下，首次将世界模型、偏好学习、理由引导和 MPC 端到端集成，无需真实环境交互即可完成训练与部署
解决的问题 / 适用场景
适用于动态未知、无法定义显式奖励函数的安全关键环境，如自动驾驶、机器人操作等场景。核心解决「无法在真实环境中试错训练」与「奖励函数难以手工设计」两重困境，通过人类在模拟器中离线反馈替代在线探索。

A Distributed Framework for Compiling and Reasoning with d-DNNF
分布式d-DNNF编译与推理框架
未明确披露
点击收起
摘要总结
知识编译（KC）通过将命题公式转换为可处理的目标语言（如d-DNNF）实现高效推理，但真实世界实例规模增大后，单机编译成为内存和时间瓶颈。本文提出dkc——首个分布式知识编译器，采用Cube-and-Conquer策略将搜索空间划分为独立子问题分发到多节点处理，有效避免了电路任务中work-stealing架构的通信开销。同时提出dreasoner分布式推理引擎，支持在分布式d-DNNF结构上进行模型计数、直接访问和均匀采样等核心推理任务，包括变量条件化下的推理。实验表明分布式架构能有效扩展，编译和查询超越最先进顺序编译器的复杂公式。代码基于开源DisCount框架扩展（d4v2 + flow-cutter），已开源。
核心创新点
  • 首次将分布式计算引入d-DNNF知识编译，提出dkc编译器，突破单机内存/时间瓶颈
  • 将Cube-and-Conquer策略从模型计数领域创新性地适配到持久化电路生成，设计全局虚拟电路（Global Virtual Circuit）抽象，避免节点间电路片段共享的通信开销
  • 提出dreasoner分布式推理引擎，首次实现在分布式d-DNNF结构上的多类型查询（模型计数、直接访问、均匀采样、条件化推理）
  • 增量式复用d4v2编译器的缓存，避免不同cube子问题间的冗余计算
解决的问题 / 适用场景
适用于大规模命题公式推理场景：约束求解、自动规划、模型计数、均匀采样等AI推理任务中，单机知识编译阶段成为瓶颈时，可通过分布式集群扩展来编译和查询超越单机能力的复杂公式。特别适合需要持久化d-DNNF结构并后续频繁查询的推理系统。

Learning Latency-Aware Orchestration for Multi-Agent Systems
学习面向延迟感知的多智能体系统编排
中佛罗里达大学
点击收起
摘要总结
论文提出 LAMaS（Latency-Aware Multi-agent System），一种延迟感知的多智能体编排框架。现有编排方法主要优化任务准确率和推理成本，但多智能体系统的端到端延迟由关键执行路径而非总算子成本决定，单纯降本不一定降延迟。LAMaS 在训练时通过带关键路径感知信用分配的约束优化学习延迟感知的执行图；在推理时补充一个轻量级控制器，根据执行过程中的实时证据自适应裁剪冗余的智能体交互。在 GSM8K、HumanEval、MATH、MMLU-Pro 四个基准上，LAMaS 相比学习型基线方法实现超过 50% 的端到端延迟降低，同时保持竞争或更优的准确率。
核心创新点
  • 首次将关键执行路径（critical path）概念引入多智能体系统编排优化，明确区分端到端延迟与总推理成本
  • 提出关键路径感知的信用分配机制，在训练阶段通过约束优化学习延迟最优的执行图结构
  • 设计轻量级推理时控制器，基于算子完成的实时输出和时间信息动态裁剪未来冗余交互，弥补训练时图结构无法利用运行时信息的缺陷
解决的问题 / 适用场景
适用于基于 LLM 的多智能体协作系统，尤其是延迟敏感的在线服务场景（如交互式编码助手、数学推理、知识问答）。解决多智能体系统因多步执行和重复调用的高延迟瓶颈，目标是让延迟自动优化而无需手工调整编排逻辑。框架具有模块化设计，可迁移至其他多智能体系统。

KV 缓存的自适应过滤：诊断与修正 LLM 推理中的结构角色偏差
未明确披露
点击收起
摘要总结
该论文系统诊断了基于注意力的 KV 缓存淘汰方法（如 H2O）在结构化输入（如嵌套 JSON）上的结构性偏差问题。研究发现，这类方法按累积注意力分数保留 token 时，会不成比例地保留分隔符、空白等非内容噪声 token，而丢弃携带答案的关键 VALUE token——在 5% 缓存预算下精确匹配准确率从 88% 骤降至 0%。作者提出一种免重训练的角色条件分配方法，在 SnapKV 的窗口分数基础上通过一个可调超参数实现角色感知的缓存分配，在 20% 以下预算下弥合了 63-98% 的 H2O 性能差距。
核心创新点
  • 首次系统诊断了基于注意力分数的 KV 缓存淘汰在 schema 密集型输入上的结构角色偏差，揭示了噪声 token 被过度保留而答案携带 token 被丢弃的根本原因
  • 通过反事实实验证明，抑制 KEY 角色 token 是最优的可部署过滤策略，而非简单地保留注意力分数最高的 token
  • 提出免重训练的角色条件缓存分配方法，结合轻量级线性角色探测模型（15 MB）与 SnapKV 窗口分数，在极低推理开销下实现角色感知缓存管理
  • 在高缓存预算下展现出微弱的去噪效果（边缘显著），表明该方法不仅能弥合性能差距，在某些设置下甚至能超越全缓存准确率
解决的问题 / 适用场景
解决 LLM 在处理嵌套 JSON、编程代码等 schema 密集型长文本时，KV 缓存压缩导致下游精确匹配任务准确率崩坏的问题。适用于需要长上下文推理且缓存资源受限的 LLM 部署场景，包括 JSON 解析、结构化数据问答、代码理解等任务。

核心创新点
  • 首次将零售预测评估从纯点误差最小化扩展为精度-稳定性权衡视角，提出了训练时稳定性正则化这一系统框架
  • 设计了融合近期需求嵌入与多维度特征（日历、价格、层级、商品、门店）的时序管道，将稳定性惩罚无缝嵌入训练过程
  • 在 M5 数据集三个规模级别上系统对比了训练时正则化与事后平滑方案，证明前者在相近稳定性收益下显著保留更多点预测精度
  • 揭示了稳定性正则化的 RMSE 代价边界（<0.72%），为运营级零售预测提供了可量化的精度-稳定性权衡参考
解决的问题 / 适用场景
解决零售需求预测中因相邻预测波动剧烈导致的供应链计划不稳定问题。适用于大规模零售场景中需将同一预测结果同时用于补货、库存、劳动力规划和运输调度等下游任务的场景，帮助企业在不牺牲预测精度的前提下获得更平滑、更可操作的预测序列。

  • 构建可扩展的基准框架，支持多编程语言、多提示模板和标准化实验流程，解决了现有基准语言覆盖窄、评价维度少的问题
  • 系统分析了 LLM 生成代码中的常见错误类型并给出具体案例，揭示模型在困难问题和低资源语言上的系统性失败及基础性错误
  • 在统一框架下同时评测开源与闭源模型，为跨模型代码生成能力对标提供了标准化比较手段
解决的问题 / 适用场景
适用于需要系统评估和对比 LLM 代码生成能力的研究人员和工程团队。解决了现有基准测试维度单一、语言覆盖窄的问题，可用于评测模型在多语言环境下的真实代码生成质量、定位薄弱环节，以及指导代码生成模型的改进方向。
