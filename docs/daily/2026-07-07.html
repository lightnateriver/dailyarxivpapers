摘要总结
该论文发现 LLM Agent 在多步任务中常陷入注定失败的轨迹，但直到失败显现前仍在持续消耗推理计算资源。作者证明失败可从 Agent 内部表征早期预测：在每轮交互中对隐藏层激活使用轻量探针，早在第一轮就能预判整个回合的最终失败。进一步提出一种可部署的中止级联机制——为每轮设置一个无分布假设的校准门控，并通过联合搜索各轮召回预算，保证最终成功回合以用户指定的全局比率通过所有门控。
核心创新点
  • 首次证明基于隐藏层激活的轻量探针可在 Agent 第一轮交互时就预测回合级失败，远优于仅依赖行为的方案
  • 提出带全局召回保证的级联中止机制，通过联合搜索各轮门控的召回预算，满足用户指定的成功回合保留率（90%-97%）
  • 在 TextCraft 任务上，Qwen-2.5-7B 和 Llama-3.2-3B 两个模型分别节省 47.1% 和 37.2% 推理计算量，最优单门控策略的近 2 倍
  • 分析了高召回目标的可认证样本复杂度，为实践者提供了其数据能否支撑特定召回承诺的理论界限
解决的问题 / 适用场景
解决 LLM Agent 在多步推理任务中无效计算浪费问题，适用于需要频繁调用 Agent 的部署场景（如代码生成、工具使用、Web 导航等），通过提前中止注定失败的回合来降低推理成本

面向推理泛化的几何自蒸馏
未明确披露
点击收起
摘要总结
论文指出，在 on-policy 自蒸馏中，教师和学生是同一模型，但教师因看到特权上下文（如完整解题 trace）而对学生产生不可靠的监督信号：教师确信的 next-token 学生尚无法自洽推理，导致蒸馏梯度在不同更新步中累积漂移，严重损害 OOD 推理性能。作者提出 GeoSD 几何自蒸馏目标函数，从两方面对抗这种漂移：Hellinger 损失按学生当前分布与教师的重叠程度缩放每个偏好信号，抑制学生尚未能支持的 token 上的拉拽；Fisher-Rao 近端惩罚项限制学生预测分布偏离近期检查点。两者在同一 next-token 分布几何中定义，并使用自然梯度更新而非参数空间更新。在数学推理 benchmark 和三种模型族、1.7B~32B 规模上，GeoSD 保持自蒸馏的 ID 增益，OOD 准确率比基础模型平均提升 5.7~8.6 个点。
核心创新点
  • 首次系统分析自蒸馏 OOD 推理退化机制：蒸馏通过从高熵状态的可选 token 上抽取概率质量来迎合教师，导致高置信度地输出错误答案，而非学到稳健的推理行为
  • 提出 GeoSD 双约束目标——Hellinger 衰减损失 + Fisher-Rao 近端惩罚，两者都在 next-token 分布流形上定义，数学共同点是自然梯度更新
  • 分布空间而非参数空间的自然梯度更新方案，从根本上改变蒸馏优化的几何结构
  • 在 1.7B（Gemma、Phi）到 32B（Qwen）多模型上验证，OOD 准确率一致提升 5.7~8.6 点，ID 增益不退化
解决的问题 / 适用场景
解决 LLM 在后训练阶段使用自蒸馏（self-distillation）时，模型对分布外（OOD）推理问题的泛化能力退化问题。适用于数学推理、逻辑推理等需要严谨链式推理的场景，确保模型在未见过的推理题型上仍能可靠作答

MoWorld：闪电级实时世界模型
未明确披露
点击收起
摘要总结
MoWorld 提出了一套从数据生成、预训练、蒸馏到推理的全栈高效世界模型框架，首次在 NPU（神经网络处理器）上实现 50 FPS 的实时交互式世界模拟，且无需高端 GPU。其核心创新在于：基于 3D 原生数据引擎构建几何一致的训练数据、渐进式课程预训练策略、高效的降噪步蒸馏算法以及混合精度并行推理框架。在 VBench-I2V 等多个基准上，MoWorld 在保持领先生成质量的同时，推理成本仅为现有世界模型的 30%-50%。
核心创新点
  • 端到端全栈协同优化：从数据引擎、课程预训练、降噪步蒸馏到推理部署，每一环节都针对效率与质量的联合优化，而非仅关注模型规模扩展。
  • 3D 原生数据引擎：区别于依赖大规模视频语料的现有方法，利用团队在 3D 视觉和几何基础模型上的积累，构建了几何一致、带精确相机姿态的世界模型训练数据，覆盖真实与合成场景。
  • 首个 NPU 原生实时世界模型：通过算子级优化、编译器感知调度和 NPU 原生执行管线，在消费级 NPU 上实现 50 FPS 实时交互，推理成本仅为现有方案的 30%-50%。
  • 自回归流匹配蒸馏（Self-Forcing Distillation）：将双向预训练模型蒸馏为因果自回归学生模型，结合历史上下文选择和教师-伪模型分布匹配，实现少步生成且保持长程一致性。
解决的问题 / 适用场景
解决现有世界模型推理速度慢、部署成本高、依赖高端 GPU 的问题，面向自动驾驶/具身智能中的实时感知规划、内容创作/影视制作的交互式场景生成、云游戏/数字孪生的实时渲染等需要低延迟闭环交互的应用场景。

摘要总结
本文研究能否在无需反复性能剖析的情况下，利用静态源码指标结合方法级执行时间来预测 Java 方法的能耗。作者对 2786 个 Java 方法进行了性能剖析，提取 33 个静态特征并测量执行时间与能耗，训练并对比了 11 种回归模型。结果表明纯静态指标预测效果极差（R² 接近 0），加入执行时间作为轻量动态输入后 R² 提升至 0.46。

  • 提出将执行时间作为轻量级动态输入补充静态指标，显著提升能耗预测准确度
  • 识别出执行时间、内部方法调用次数和圈复杂度是最主要的能耗预测因子
解决的问题 / 适用场景
解决软件开发早期难以评估代码能耗的问题，适用于 Java 应用开发中的绿色软件设计、代码重构前的能耗评估，以及大型软件系统的能耗优化决策。

摘要总结

  • 发现潜在推理步骤对最终答案的因果贡献随训练而衰减，且该衰减在二选一与开放式解码上呈现分岔轨迹
解决的问题 / 适用场景
