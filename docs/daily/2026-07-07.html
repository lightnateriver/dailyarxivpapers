未明确披露
点击收起
摘要总结
论文发现 LLM Agent 在执行多步任务时，许多失败的轨迹在早期就已可预测——利用轻量级逐轮探针（per-round probe）读取 Agent 内部隐藏状态，能在第一个交互回合就预判整轮回是否失败，而仅观测行为的评分器在此时几乎不可能判断。作者设计了一套实用的中止级联机制：每轮设置一个无分布假设的校准门控，联合搜索各轮召回预算，使最终成功的回合以用户指定的全局概率通过所有门控。在 TextCraft 上，该级联在 90% 召回目标下为 Qwen-2.5-7B 和 Llama-3.2-3B 分别节省 47.1% 和 37.2% 推理计算量，是单门控策略的 1.6-1.7 倍。仅依赖行为信号的级联效果减半，且向探针加入行为特征无额外收益，说明隐藏状态已包含全部可预测信息。
核心创新点
  • 首次从 LLM Agent 内部隐藏激活中早期预测回合失败，首轮即可显著胜于基于行为的分类基线
  • 提出多轮级联中止机制，通过联合搜索各轮召回预算保证用户指定的全局成功率，解决假中止风险累积问题
  • 系统证明隐藏状态编码了比可观测行为更丰富、更早的失败信号，行为特征在探针上无额外增益
  • 刻画了认证高召回目标所需的样本复杂度，为实际部署提供可证明的数据量下限指导
解决的问题 / 适用场景
解决 LLM Agent 在多步任务中明知失败仍持续消耗推理计算资源的问题，适用于任何 LLM Agent 推理部署场景（例如代码生成、Web 浏览、工具使用等），可大幅降低无效推理成本，提升系统吞吐量

静态指标不足：用执行时间预测 Java 方法的能耗
未明确披露
点击收起
摘要总结
本文研究 Java 方法级别的能耗预测问题，探究静态源代码指标辅以方法级执行时间能否有效估计方法能耗。研究对 2,786 个 Java 方法进行了性能剖析，提取 33 个静态特征，并训练比较了 11 种回归模型。结果表明，仅凭静态源码指标预测性能极差（平均 R² 接近 0），引入执行时间作为轻量级动态输入后预测精度显著提升（R² 最高达 0.46），其中执行时间、内部方法调用次数和圈复杂度是能耗的最强预测因子。
核心创新点
  • 系统评估了 33 个静态代码指标在 Java 方法级能耗预测上的极限，实证其单独预测能力接近于零
  • 提出将方法级执行时间作为轻量级动态输入，使预测 R² 从近 0 提升至 0.46，证明了少量动态信息的巨大价值
  • 识别出执行时间、内部方法调用次数和圈复杂度为能耗最重要的三个预测因子
解决的问题 / 适用场景

仅看最终检查点不够：沿训练轨迹分析潜在推理的忠实度
未明确披露
点击收起
摘要总结
本文研究潜在推理（latent reasoning）方法中推理步骤对最终答案的因果忠实度，并追踪忠实度在训练过程中的演化轨迹。通过在输入上施加可验证的反事实编辑、对潜在推理步骤进行噪声消融激活补丁，作者发现在收敛检查点看似不忠实的潜在推理方法，其训练轨迹可能截然不同；潜在推理步骤对最终答案的因果贡献随训练整体衰减，且该趋势在二选一和开放式解码两种答案格式上呈现分化。
核心创新点
  • 首次沿训练轨迹追踪潜在推理忠实度的演化，而非仅分析收敛检查点，揭示了收敛时相似的不忠实行为可能源自截然不同的训练轨迹
  • 发现潜在推理步骤的因果贡献随训练过程整体衰减，且该衰减趋势在输出层面表现为反事实编辑下的行为分歧
  • 揭示答案格式（二选一 vs 开放式解码）对潜在推理忠实度训练轨迹有显著影响，两种格式上因果贡献的变化方向相反
解决的问题 / 适用场景
解决潜在推理方法中隐藏状态的不透明性和可信性问题，适用于需要可解释推理的 LLM 应用场景，如安全关键系统的推理可信度评估、模型训练过程中的质量监控，以及设计更忠实的推理架构。

点击收起
摘要总结
探究扩散模型后剪枝重训练能否被步数蒸馏替代。发现直接替换失败——剪枝后的 EDM2-XS 检查点直接进行 SiDA 蒸馏会产生不可用的样本。为此引入一个短期的教师对齐修复阶段作为剪枝与步数蒸馏之间的桥梁，使剪枝后的生成器在含噪真实图像隐空间上与教师对齐，再将修复后的检查点交给一步蒸馏。在 ImageNet-512 上，20% 剪枝的一步生成器（98.826M 参数、1 次网络评估）FID 达到 3.12，优于原版 124.713M 参数、63 次评估的 FID 3.53。
核心创新点
  • 首次系统揭示扩散模型后剪枝重训练直接替换为步数蒸馏会导致失败
  • 提出短期的教师对齐修复阶段作为剪枝与蒸馏之间的桥梁，解决剪枝后蒸馏梯度不可用问题
  • 剪枝加单步蒸馏的组合在参数量和推理步骤上同时大幅压缩，FID 反而优于原模型
解决的问题 / 适用场景
解决扩散模型推理由大网络体量（参数多）和多步去噪（计算多）造成的双重成本问题，适用于扩散模型在资源受限设备上的部署与低延迟推理场景
