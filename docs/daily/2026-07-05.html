未明确披露（作者来自多机构：Yaozu Wu等14位作者，含Philip S. Yu等）
点击收起
摘要总结
提出HAS-Framework，一种基于图的框架，将人类和LLM智能体视为具有显式角色、权限、通信路径和行动权威的一等参与者。在此基础上构建HAS-Bench基准，在可配置的人类参与下（涵盖自主性级别、交互渠道和个性策略），系统评估人机协作系统的任务结果和过程级协作行为（包括澄清质量、反馈利用、控制校准、安全性、主动性和交互成本）。在六个领域上的实验表明：人类参与能显著提升任务完成率和故障恢复能力，但收益取决于人类输入的时间、方式和执行者。
核心创新点
  • 提出图结构人机协作框架HAS-Framework，将人类与LLM智能体同等地建模为一等参与者，并显式定义角色、权限、通信路径和行动权威，这是首个对人机协作进行结构化表征的形式化框架
  • 构建HAS-Bench多维评测基准，同时覆盖任务结果和过程级协作行为指标（澄清质量、反馈利用、控制校准、安全性、主动性和交互成本），超越了仅关注任务完成率的传统评测方法
  • 引入可配置的人类参与机制——跨自主性级别（从完全被动到完全主动）、交互渠道（文本/语音/图形界面）和个性策略——系统揭示人类参与的时间、方式和执行者如何影响协作收益
  • 在六个不同领域（包括复杂推理、决策制定等）上进行大规模实验，提供关于人机协作中人类何时介入最有效的第一个系统性实证分析
解决的问题 / 适用场景
适用于需要人类与LLM智能体深度协作的应用场景，如客服系统、医疗辅助诊断、教育辅导、创意协作等。解决的核心问题是：在哪些环节、以何种方式引入人类参与才能最大化人机协作系统的整体性能，以及如何系统性地评估这种协作质量。为设计人机协作AI系统提供了可量化的评估方法论。

Prompt-to-Paper: Agentic AI System for Bioinformatics
Prompt-to-Paper：面向生物信息学的智能体式AI论文生成系统
巴基斯坦国立科技大学（NUST），跨学科工程与科学学院（SINES），伊斯兰堡
点击收起
摘要总结
提出Prompt-to-Paper（RLEv4），一个多智能体论文生成框架，旨在解决现有AI自动撰稿系统的三个关键缺陷：生成内容无法被可验证文献确定性地支撑、实验结果通常被编造而非真实执行、缺乏标准化的多维质量评估框架。该系统通过三项核心创新闭环解决这些问题：确定性RAG管线（含章节感知相关性评分和雪球引用扩展，将每一句声称锚定在60-100篇可验证论文中）、自主编码智能体（真实执行计算生物学实验，输出真实数值而非合成数据）、以及八维自动质量评分器（含幻觉惩罚）。质量驱动的改进循环每10轮触发一次深度研究周期——重跑实验、重构稿件。在五个生物信息学案例上验证，生成符合投稿格式的PDF，零越界引用，改进循环平均提升质量17.96分（满分100），每篇成本约0.31美元。
核心创新点
  • 确定性RAG管线：采用章节感知相关性评分（按abstract 0.30、methods 0.25、results 0.20等加权）和雪球引用扩展（两轮迭代将种子论文的引用/被引扩展至60-100篇），配合SPECTER2嵌入和PageRank排序，确保每句声称都有可追溯的真实文献支撑
  • 自主编码智能体执行真实实验：自动编写、调试并执行Python脚本（含多序列比对、Jukes-Cantor距离计算、5000次置换检验等），将真实数值注入稿件每一节，彻底杜绝合成/编造结果问题；配备语法预检（ast.parse）和12次真实执行尝试机制
  • 八维混合质量评分器：结合LLM-as-judge G-Eval（55%权重，用deepseek-v4-pro评测新颖性、贡献、可靠性、表现力、可复现性、文献根基等8个维度，z-score对标ICLR参考统计）、实体网络指标（25%）、启发式表面特征（20%）；内置确定性幻觉惩罚（越界引用标记/数值不一致检测）
  • 深度研究循环：每10轮改进迭代触发一次完整的研究周期——Leader模型读取全文识别科学缺口→规划新实验架构→编码智能体改进并执行代码→Leader用更强输出重写全文→仅当评分不退化时替换；配合三级行动路由（add_analysis→gather_evidence→rewrite）和压缩记忆机制，避免无效重复修改
解决的问题 / 适用场景
解决的是AI自动生成学术论文过程中的内容虚构、实验造假和质量不可评估三大痛点。特别适用于生物信息学领域的计算实验型论文（序列分析、结构预测、系统发育等），可推广到其他需要文献综述+实验验证+图表生成的科研场景。为自动化科学发现和论文撰写提供了可验证、可复现、可评分的技术路径，每篇成本仅0.31美元，具有高性价比的实际应用潜力。

测量多步 LLM Agent 中 Harness 引发的信念分歧

点击收起
摘要总结
本文发现，评估 LLM Agent 时使用的 harness（控制 agent 可见信息、可选动作、故障修复、状态验证和日志记录的框架）会改变 agent 的多步信念，即使任务、环境和基础 LLM 都固定不变。作者引入信念展开诊断方法，在进度、风险、可恢复性、约束、故障模式、不确定性等维度上提取结构化的 K 步轨迹，并定义了跨 harness 的信念分歧度量。实验表明，不同 harness 设计（如屏蔽动作、压缩修复、选择性验证）往往在保持最终任务成功率的同时，显著改变了驱动后续决策的信念。

  • 引入跨 harness 信念分歧分解（到达项 + 增长项）及 BIWM 协议，实现了不同 harness 下信念轨迹的对齐与规范

未明确披露
点击收起
摘要总结
检索增强生成(RAG)虽能减少幻觉但无法彻底消除，现有检测器仅输出整体答案级分数，无法定位到具体句子或指出原因。本文提出GASP，一种句子级幻觉检测方法，通过衡量每个答案句的似然对检索证据的依赖程度（称为接地敏感性）来判定是否幻觉。具体做法是固定答案，分别在完整上下文、无上下文和逐段移除上下文下重新评分，计算对数似然下降和Jensen-Shannon散度。接地句在移除支持段落时似然骤降，而幻觉句几乎不受影响。在RAGTruth、TofuEval、RAGBench三个基准上使用Qwen2.5-0.5B/1.5B和SmolLM2-1.7B评估，RAGTruth上响应级AUC约0.73、句子级AUC约0.67，显著优于困惑度、长度、全上下文NLI和自一致性基线。无需训练数据的阈值版即可匹配有监督分类器。
核心创新点
  • 提出接地敏感性这一新指标，通过测量句子似然对检索证据的依赖程度来区分接地句与幻觉句，无需额外标注数据
  • 将解码过程建模为随机非线性迭代函数系统(RNIFS)，从理论上解释了移除支持段落时接地句似然骤降而幻觉句几乎不受影响的对比现象
  • 设计三条件评分策略（完整上下文、无上下文、逐段移除），组合对数似然下降与JSD进行句子级幻觉检测
  • 无训练版本（仅靠接地特征阈值）即可匹配有监督分类器性能，实现零标注数据下的幻觉检测
解决的问题 / 适用场景
适用于RAG系统的输出质量保障场景，特别是需要细粒度定位哪些句子缺乏证据支撑的长文本生成任务；也可用于RAG管线的质量监控和调试，帮助开发者发现检索环节或生成环节的薄弱之处
