Dreamer-CPC: Message Learning with World Models for Decentralized Multi-agent Reinforcement Learning
Dreamer-CPC：基于世界模型的去中心化多智能体强化学习消息学习
京都大学，立命馆大学
点击收起
摘要总结
提出 Dreamer-CPC，一种去中心化的基于模型的多智能体强化学习方法。它将集体预测编码（CPC）引入 DreamerV3 的世界模型中，让每个智能体独立维护一个世界模型和消息模块，从世界模型的隐状态中推断并交换携带历史信息的消息，而非仅依赖当前观测。在 Observer 和新提出的 CatchApple 环境中，Dreamer-CPC 在部分可观测条件下显著超越基于当前观测生成消息的 IPPO-CPC 和无通信基线，CatchApple 中回合回报达到 IPPO-CPC 的 4-5 倍。
核心创新点
  • 将集体预测编码（CPC）整合进 DreamerV3 的世界模型框架，使智能体从隐状态序列中学习消息表示，而非从单帧观测编码消息
  • 每个智能体独立维护世界模型和消息模块，实现完全去中心化训练与执行，无需集中式梯度传播或参数共享
  • 设计了 CatchApple 任务环境，模拟智能体观测临时缺失的协作场景，证明基于世界模型动态的通信能有效处理当前观测不足以决策的困境
解决的问题 / 适用场景
解决部分可观测环境下多智能体协同决策中消息仅含当前观测、缺乏历史信息累积的问题。适用于分布式机器人协作、多智能体感知信息共享、以及局部观测暂时缺失等去中心化场景。

PerfAgent：基于 Profiler 引导的迭代精炼仓库级代码优化
MIT，伦斯勒理工学院，IBM Research，密歇根州立大学
点击收起
摘要总结
提出了 PerfAgent，一个 profiler 引导、verifier-in-the-loop 的仓库级代码优化工作流。它通过采样 profiler 精确定位性能热点（包括跨抽象层和原生扩展内的瓶颈），引入目标驱动的循环控制器拒绝智能体的首个通过补丁并持续迭代优化，并配合选择性测试阶段防止破坏边缘情况。在 GSO 和 SWE-fficiency-Lite 两个基准上，PerfAgent 将 OpenHands + GPT-5.1 的专家级补丁率从 19.6% 提升至 39.2%（GSO）、从 26% 提升至 74%（SWE-fficiency-Lite），且以更低成本超过 oracle best-of-five 基线。
核心创新点
  • 引入采样 profiler 对运行负担进行热点摘要（位置、调用上下文、时间占比），使智能体能发现隐藏在抽象层和原生扩展背后的真实瓶颈
  • 设计目标驱动的循环控制器：拒绝首个通过补丁，每次提交后重建、重验证、重分析，保留最快正确补丁并在多轮迭代中持续优化
  • 选择性测试阶段确保性能优化不破坏边缘情况和下游代码路径，解决现有方法因测试不充分导致补丁在非目标负载上静默失效的问题
  • 系统性地证明优化收益来自更好的反馈机制而非增加测试时采样次数，以更低成本超越 oracle best-of-five 基线
解决的问题 / 适用场景
解决现有 LLM 代码智能体在仓库级代码优化中浅尝辄止、遗漏深层瓶颈、测试不充分的问题。适用于真实仓库的性能优化任务，特别是涉及多语言、跨抽象层、含原生扩展的复杂代码库优化场景。

CHMAS: A Coupled Hierarchical Framework for Multi-Agent Reinforcement Learning
CHMAS：面向多智能体强化学习的耦合分层框架
加利福尼亚大学河滨分校
点击收起
摘要总结
本文提出CHMAS（耦合分层多智能体系统）框架，将多智能体决策分解为集中式战略规划和分布式战术执行两个层次，并建立双向信息流机制。战略层每T步整合全局状态生成指导动作，战术层在战略指导和局部邻域观测下执行分布式策略。与现有单向控制的分层方法不同，CHMAS引入耦合系数λ实现的反馈机制，使战术累积奖励反向影响战略目标，确保战略规划不脱离战术可行性。理论分析证明异步更新方案在标准假设下实现O(log K/√K)的收敛率，在多智能体觅食场景中验证了空间分区探索策略的成功学习。
核心创新点
  • 提出双向耦合的分层MARL架构，通过反馈机制（耦合系数λ）让战术层经验反向调整战略层目标，打破传统分层强化学习中单向控制的局限
  • 设计了异步更新协议（战略参数每N_f个战术episode更新一次），让战术策略可在战略更新间隔内收敛至准稳态，缓解分层学习的非平稳性问题
  • 同时给出了通用的双层形式化建模和易于处理的加性近似版本，并证明了异步方案在K次战略更新后达到O(log K/√K)收敛率
  • 在多智能体觅食任务中展示了智能体自动学会空间分区探索策略，验证了分层耦合在复杂协作场景中的有效性
解决的问题 / 适用场景
解决多智能体系统中全局协调与局部执行在不同时间尺度上难以平衡的问题。适用于需要多机器人/多智能体协同完成探索、搜索救援、仓储物流等任务的场景，特别是需要长期战略规划和短期战术执行紧密结合的应用。

摘要总结
提出AutoGlue，一个层次化多智能体框架，用于自动生成行为驱动开发(BDD)中的Java胶水代码(glue code)。该框架采用"行为优先"工作流，将行为理解、上下文检索和代码生成三个环节解耦：Behavior Interpreter从场景上下文中推导步骤意图，Developer agent检索相关BDD工件和项目代码后生成最终胶水代码。在8个开源Java项目的1307个步骤上评估，相比少样本提示，API F1提升58.7%，CodeBLEU提升43.7%；46.1%的步骤生成了可直接使用的胶水代码，其余大部分仅需微调。
核心创新点
  • 首次将LLM用于BDD胶水代码自动生成任务，填补了该方向的研究空白——此前LLM代码生成研究主要聚焦函数级或类级代码而非胶水代码
  • 设计层次化多智能体架构(AutoGlue)，将行为理解、项目上下文检索、代码生成分离为独立子智能体，避免单模型在同一提示中处理异构信息的认知过载
  • 提出行为优先(behavior-first)的解耦工作流：先由Behavior Interpreter解析步骤在场景中的语义意图，再由Developer agent感知项目代码结构后生成实现，使每一步的推理目标明确
解决的问题 / 适用场景
解决BDD实践中胶水代码手工编写和维护成本高的问题——当需求演化、场景步骤新增或修改时，开发人员需要同时理解自然语言行为和项目代码才能编写匹配的胶水代码。适用于采用Cucumber等BDD框架的Java项目团队，尤其是需求频繁变动的敏捷开发场景，可显著降低BDD自动化测试的维护负担。
