However, scaling and orchestrating such agents effectively remains challenging, due to the difficulty of coordinating parallel proof search while keeping intermediate claims organized and reliable. In this paper, we propose Danus, an orchestration system for research-level mathematical reasoning centered on a shared fact graph as a global memory-management mechanism. Danus consists of a main agent that performs planning and coordination, multiple worker agents that carry out proof search in parallel, and a stateless verifier that checks proposed mathematical claims before they are admitted into the fact graph. Each verified fact is stored together with its proof and logical dependencies, allowing the system to build long arguments incrementally while keeping the shared proof state organized. The main agent periodically summarizes the evolving proof state, redirects workers across promising directions, and supports interaction with human mathematicians through progress reports. We evaluate Danus through six research-level case studies in algebraic geometry, singularity theory, and combinatorics, illustrating how the fact-graph memory mechanism enables Danus to construct long, detailed mathematical proofs. Our results suggest that fact-graph-based orchestration provides an effective route toward scaling mathematical reasoning agents for long-horizon research problems. 摘要总结
这篇论文提出了面向研究级数学推理任务的编排系统 Danus，用共享事实图作为全局记忆来组织证明过程。系统由负责规划协调的主智能体、并行执行证明搜索的多个工作智能体，以及用于校验数学断言的无状态验证器组成。被验证通过的事实会连同证明及逻辑依赖一起写入事实图，从而支持逐步构建长链条证明。作者通过多个研究级数学案例说明，这种基于事实图的编排方式有助于扩展长时程数学推理能力。

  • 将已验证事实与其证明和逻辑依赖共同存储，支持长证明过程的增量式构建与追踪。
  • 支持对不断演化的证明状态进行周期性总结，并据此重定向搜索方向及生成人类可读进展报告。

这篇论文关注长时程智能体训练中的在线策略蒸馏效率问题。作者指出传统 OPD 在长轨迹任务中存在两类低效：尾部轮次会浪费训练时间，而轨迹级 KL 目标又容易让浅层 token 主导训练。为此，论文提出 TurnOPD，通过轮次级预算控制来决定 rollout 深度，并逐步把监督重点从 token 级转向更均衡的轮次级监督。实验显示，该方法在相同训练时间预算下取得了更好的验证表现。

摘要总结
该论文研究将记忆读写放入语言 Agent 的每一步 observe-reason-act 循环中，而非每轮仅查询一次外部记忆。作者认为，阻碍这一范式的关键不是模式本身，而是网络化存储带来的高延迟；当记忆存储迁移到进程内后，检索延迟可大幅下降，从而使其更像扩展工作记忆。实验展示了这种 in-process memory 能在受限上下文窗口下改善回忆能力，并指出新的主要瓶颈转移到了 embedding。
核心创新点
  • 提出将 memory 放入 agent 每一步循环内部的 in-process retrieval 范式。
  • 从延迟角度重新解释 memory 设计，认为限制来自存储位置而非 in-loop 模式本身。
  • 通过对比不同记忆响应延迟，分析延迟上升如何导致冗余动作增加。
  • 指出在进程内存储足够快时，系统瓶颈会从检索转移到 embedding 过程。
解决的问题 / 适用场景
适用于需要长期记忆、受上下文窗口限制且需频繁读写状态的语言智能体系统，如任务执行 Agent、对话 Agent 和带检索的推理 Agent。它主要解决的是传统外部记忆延迟高，难以支持细粒度环内调用的问题。
