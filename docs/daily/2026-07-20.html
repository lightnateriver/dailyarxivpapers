点击收起
摘要总结
现代GPU加速器平台依赖跨越硬件、固件、管理软件和编排层的多层自愈流水线来保障可用性。当故障跨层传播时，可能绕过检测、污染诊断或触发冲突修复，但传统故障注入测试仅针对单层独立进行。本文提出ADA-ST，一种基于加权故障传播图引导跨层场景选择的自适应故障注入方法，并在超大规模运营商的三代平台（Alpha/Beta/Gamma）上构建四层传播图，72,550张维修工单中49%涉及跨层传播。实验表明现有静态测试仅覆盖20-25%的故障传播边，ADA-ST通过迭代活动引导场景选择，在10/12/9轮内达到全边覆盖率。
核心创新点
  • 提出ADA-ST自适应故障注入框架，使用加权故障传播图建模跨层故障传播路径，替代传统单层隔离测试方法

西安交通大学 + 北京大学
点击收起
摘要总结

核心创新点
  • 揭示在线策略自蒸馏的初始化瓶颈问题：从冗长模型启动时，KL损失作用于噪音/冗余前缀，简洁教师只能提供局部修正
  • 提出两阶段BIRD框架：先通过简洁指令采样+提示交换SFT引导出默认简洁推理行为，再进行在线策略自蒸馏

  • 在Qwen3全系列和DeepSeek-R1-Distill-Llama上验证，在显著缩短推理长度的同时保持甚至提升准确率
解决的问题 / 适用场景
解决大型推理模型推理开销过大问题，适用于对推理延迟和计算成本敏感的LLM部署场景，特别适合数学推理、竞赛级问题求解等需要长链推理但希望控制token消耗的应用

PagedWeight：基于动态质量感知权重量化的高效 MoE 大模型推理服务
伊利诺伊大学厄巴纳-香槟分校 (UIUC)
点击收起
摘要总结
PagedWeight 是一种面向 MoE 大语言模型推理服务的动态权重量化管理方法。在 KV cache 密集型服务场景中，MoE 模型权重与 KV cache 之间存在 GPU 内存竞争；PagedWeight 在运行时动态量化 MoE 权重，在专家权重精度与 KV cache 大小之间自动平衡，并显式建模精度、内存、吞吐量三者间的复杂权衡。相比 FP16 基线，PagedWeight 在保持等效精度的同时节省高达 72.0% GPU 内存、吞吐量提升 1.94 倍；相比现有量化方法，在相似内存预算下质量提升达 39.3%。

  • 提出 PagedWeight 运行时动态 MoE 权重量化管理机制，首次显式建模并导航精度-内存-吞吐量三者的复杂权衡
  • 在 KV cache 密集推理场景中动态调整专家权重的量化精度，自适应适配可用 KV cache 空间
  • 相比 FP16 实现 72% GPU 内存节省和 1.94 倍吞吐量提升且保持精度等价，在同等内存预算下质量优于现有量化方法最多 39.3%

Every Microsecond Matters: Achieving Near Speed-of-Light Latency in GPU Collectives
每一微秒都很重要：GPU 集合通信达到近光速延迟
NVIDIA, ETH Zurich
点击收起
摘要总结
本文针对 GPU 集合通信的延迟优化展开研究。长上下文解码密集的 LLM 推理中，大量小消息集合操作位于 token 生成的关键路径上，微秒级开销也会直接影响性能与成本。作者系统分析了现有低延迟技术，组合出无需全局内存屏障的 barrier-free 方案，基于 NCCL device-side API 开发了低延迟通信原语，并实现了新型 AllReduce 算法。微基准测试显示，小中消息延迟降至硬件速度下限（SoL）的 7% 以内。集成到 vLLM 后显著降低 inter-token latency，同时加速了 cuSOLVERMp 传统 HPC 负载。
核心创新点
  • 系统识别全局内存屏障是 GPU 集合延迟的关键来源，提出 barrier-free 同步设计范式，组合 LL、哨兵同步、双缓冲等技术彻底消除屏障开销
  • 在 NCCL device-side API 之上设计并实现了 buffer-centric 低延迟通信 API 层，封装 LL/sentinel 等多种同步模式，支持快速原型化自定义集合 kernel
  • 提出 novel two-shot LL128 atomic AllReduce 算法，利用 vectorized atomic 和 cache-line 级原子加法，仅付出约 1.5-3% 带宽代价即实现近 SoL 延迟
  • 在 GB200 NVL72 上验证，4 GPU 配置下小消息 AllReduce 延迟从 11.0μs（NCCL ring）降至 2.37μs，vLLM Llama-3.1-70B 推理 ITL 降低 8.7%，每 μs 延迟消除约节省 0.9% 推理成本
解决的问题 / 适用场景
解决多 GPU 推理系统中小消息集合通信的延迟瓶颈，尤其适用于长上下文、小 batch 的 decode-heavy LLM 推理场景，亦可加速科学计算中的频繁小规模全局规约操作。已有开源代码。

FSZ: Breaking the Prediction-Throughput Trade-off in GPU Lossy Compression
FSZ：打破 GPU 有损压缩中预测与吞吐量的权衡
University of South Florida
点击收起
摘要总结
现有快速 GPU 有损压缩器采用纯 GPU 单 kernel 设计实现了高吞吐，但由于使用固定一阶预测器对独立块进行预测，压缩率受限。本文提出 FSZ，在单个 CUDA kernel 内通过三项相互增强的算法创新同时提升压缩率和吞吐量：跨块预测状态传递（消除 7/8 的边界残差）、每瓦片自适应多阶预测与中心化、以及单趟四路评估（利用有限差分的数学性质从单次数据读取中评估所有变体）。在 NVIDIA GH200 上使用 8 个真实科学数据集验证，压缩率比 cuSZp-P 最高提升 10.95 倍，比 SOTA cuSZp-O 最高提升 2.92 倍，且无吞吐损失。
核心创新点
  • 提出跨块预测状态传递机制，在 256 元素瓦片内跨块边界携带 Lorenzo 预测状态，消除 7/8 的边界残差，大幅降低编码比特率
  • 提出每瓦片自适应多阶预测与中心化策略，从一阶、二阶及中心化变体中自适应选择最优压缩方案，无需全局调参
  • 提出单趟四路评估方法，利用有限差分的数学性质仅需单次数据读取即可并行评估所有预测变体，将丰富预测嵌入原有带宽预算内
  • 在 GH200 上达到的最高平均吞吐量（压缩 676 GB/s、解压 785 GB/s）超过所有现有评测压缩器，同时压缩率等指标的提升不牺牲吞吐
解决的问题 / 适用场景
解决 GPU 上有损压缩中预测质量与吞吐不可兼得的根本矛盾，适用于科学计算/HPC 领域的大规模浮点数据压缩场景（如气候模拟、粒子物理、结构力学），可大幅节省存储与传输带宽。已被 SC26 接收。
