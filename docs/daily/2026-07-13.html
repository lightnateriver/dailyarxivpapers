Secret Scanner Agent：从非结构化文档中提取密钥及其访问上下文
未明确披露
点击收起
摘要总结
这篇论文提出 Secret Scanner Agent（SSA），用于从邮件、聊天记录、工单和事故笔记等非结构化泄露文档中，同时提取凭据秘密及其可访问的目标对象（如账号、租户、端点、数据库或云资源）。作者认为传统基于正则或分类器的方法更适合结构化代码场景，难以处理被拆分、重排或与访问目标相距较远的凭据。SSA 通过多智能体协作，不仅定位 secret，还返回其对应“door”及支持证据，使结果更适合应急响应和后续处置。
核心创新点
  • 提出面向非结构化泄露文档的多智能体秘密扫描框架，同时抽取 secret 与其对应的访问对象“door”。
  • 设计了偏重召回的检测 agent 与负责过滤误报、补全上下文的审查 agent 协同机制。
  • 构建了覆盖 23 类 secret 和多种文档格式的合成评测基准，并采用程序匹配、LLM 评审和人工复核三阶段评测流程。
  • 将传统“只找密钥字符串”的检测结果扩展为包含 secret、door 与证据的可执行安全发现。
解决的问题 / 适用场景
适用于安全应急响应、泄露排查、合规审计和内部安全运营场景，尤其适合处理邮件、聊天、工单、事故记录等非结构化文本中的凭据泄露问题。它解决的不仅是“有没有泄露 secret”，还包括“该 secret 能打开什么系统入口”的定位问题。

摘要总结
这篇论文研究在代码生成任务中，向多模态大语言模型同时提供题目文本和程序流程图，是否能够提升生成效果。作者基于 AtCoder 题目，从示例解答代码生成流程图，并将其与题目描述一起输入 GPT-4o 进行实验。结果表明，加入流程图能够带来性能提升，同时流程图细节越充分，效果通常越好；与 few-shot 方法相比，流程图也表现出稳定的辅助作用。
核心创新点
  • 首次系统探索将程序流程图作为视觉输入，引入多模态大模型代码生成任务中的效果。
  • 构建了由示例解答代码生成流程图并与题目文本联合输入模型的实验范式。
  • 分析了不同抽象程度流程图对代码生成效果的影响，发现更丰富的流程信息通常更有帮助。
  • 将流程图辅助与 few-shot 学习进行对比，讨论了两类提示增强方式的相对作用。
解决的问题 / 适用场景
适用于编程竞赛题求解、教育编程辅助、自动代码生成以及需要把算法流程显式可视化后再交给多模态模型理解的场景。特别适合输入本身包含过程结构、控制流或算法步骤信息的代码生成任务。
